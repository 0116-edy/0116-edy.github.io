<header>

<h1>Problema Resolvido: Site Mostrando Código em Vez de Renderizar</h1>

<p>Soluções para quando o GitHub Pages exibe o código HTML em vez de mostrar o site</p>

</header>

<div class="problem-section">

<h2>Por que isso acontece?</h2>

<p>O GitHub Pages pode exibir o código em vez do site renderizado por vários motivos:</p>

<ul>

<li>Arquivo não salvo com extensão <strong>.html</strong></li>

<li>Configuração incorreta do GitHub Pages</li>

<li>Arquivo não nomeado como <strong>index.html</strong></li>

<li>Problemas com o conteúdo HTML</li>

</ul>

</div>

<div class="solution-section">

<h2>Soluções para o problema</h2>

<div class="step">

<h3><span class="step-number">1</span> Verifique a extensão do arquivo</h3>

<p>Certifique-se de que seu arquivo tenha a extensão <strong>.html</strong>.</p>

<p>Às vezes, o arquivo pode ser salvo como <strong>index.html.txt</strong> sem que você perceba.</p>

</div>

<div class="step">

<h3><span class="step-number">2</span> Ative o GitHub Pages</h3>

<p>No seu repositório, vá em <strong>Settings > Pages</strong>.</p>

<p>Em <strong>Branch</strong>, selecione <strong>main</strong> ou <strong>master</strong> e clique em <strong>Save</strong>.</p>

</div>

<div class="step">

<h3><span class="step-number">3</span> Nomeie o arquivo como index.html</h3>

<p>O arquivo principal deve chamar-se <strong>index.html</strong> para ser reconhecido como página inicial.</p>

</div>

<div class="step">

<h3><span class="step-number">4</span> Espere alguns minutos</h3>

<p>O GitHub Pages pode levar alguns minutos para processar e publicar seu site.</p>

<p>Atualize a página após 5-10 minutos.</p>

</div>

<div class="step">

<h3><span class="step-number">5</span> Verifique a estrutura do HTML</h3>

<p>Seu arquivo HTML deve ter a estrutura básica correta:</p>

<code>&lt;!DOCTYPE html&gt;

&lt;html lang="pt"&gt;

&lt;head&gt;

&lt;meta charset="UTF-8"&gt;

&lt;meta name="viewport" content="width=device-width, initial-scale=1.0"&gt;

&lt;title&gt;Título do Site&lt;/title&gt;

&lt;/head&gt;

&lt;body&gt;

&lt;!-- Conteúdo do site --&gt;

&lt;/body&gt;

&lt;/html&gt;</code>

</div>

</div>

<div class="tip">

<h3>Dica Importante</h3>

<p>Se você copiou e colou o código, verifique se não há caracteres especiais ou problemas de formatação.</p>

<p>Às vezes, editores de texto podem adicionar formatação invisível que interfere no HTML.</p>

</div>

<div class="success">

<h3>Verifique se está funcionando</h3>

<p>Depois de aplicar essas soluções, acesse seu site novamente:</p>

<p><strong>https://seuusuario.github.io</strong></p>

<p>Se ainda não funcionar, tente limpar o cache do navegador (Ctrl+F5).</p>

</div>

<h2>Configuração Avançada (se necessário)</h2>

<div class="step">

<h3>Criar arquivo de configuração</h3>

<p>Às vezes, é necessário adicionar um arquivo de configuração para garantir que o GitHub Pages reconheça seu site:</p>

<p>Crie um arquivo chamado <strong>.nojekyll</strong> (sem extensão) na raiz do seu repositório.</p>

<p>Este arquivo diz ao GitHub para não usar o processador Jekyll, que às vezes causa problemas com sites HTML simples.</p>

</div>

<div class="step">

<h3>Verificar URL correta</h3>

<p>Certifique-se de que está acessando a URL correta:</p>

<p><strong>https://seunomedeusuario.github.io</strong> (não é https://github.com/seunomedeusuario/seunomedeusuario.github.io)</p>

<p>Muitas pessoas confundem a URL do repositório com a URL do site publicado.</p>

</div>

<div class="tip">

<h3>Último recurso</h3>

<p>Se nada funcionar, tente estas alternativas:</p>

<ol>

<li>Exclua e recrie o repositório</li>

<li>Use o Netlify Drop (arraste e solte o arquivo HTML)</li>

<li>Use o Vercel para implantação</li>

</ol>

<a href="https://app.netlify.com/drop" class="btn" target="_blank">Usar Netlify Drop</a>

</div>
