Problema Resolvido: Site Mostrando Código em Vez de Renderizar

Soluções para quando o GitHub Pages exibe o código HTML em vez de mostrar o site

Por que isso acontece?

O GitHub Pages pode exibir o código em vez do site renderizado por vários motivos:

  • Arquivo não salvo com extensão .html
  • Configuração incorreta do GitHub Pages
  • Arquivo não nomeado como index.html
  • Problemas com o conteúdo HTML

Soluções para o problema

1 Verifique a extensão do arquivo

Certifique-se de que seu arquivo tenha a extensão .html.

Às vezes, o arquivo pode ser salvo como index.html.txt sem que você perceba.

2 Ative o GitHub Pages

No seu repositório, vá em Settings > Pages.

Em Branch, selecione main ou master e clique em Save.

3 Nomeie o arquivo como index.html

O arquivo principal deve chamar-se index.html para ser reconhecido como página inicial.

4 Espere alguns minutos

O GitHub Pages pode levar alguns minutos para processar e publicar seu site.

Atualize a página após 5-10 minutos.

5 Verifique a estrutura do HTML

Seu arquivo HTML deve ter a estrutura básica correta:

<!DOCTYPE html> <html lang="pt"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Título do Site</title> </head> <body> <!-- Conteúdo do site --> </body> </html>

Dica Importante

Se você copiou e colou o código, verifique se não há caracteres especiais ou problemas de formatação.

Às vezes, editores de texto podem adicionar formatação invisível que interfere no HTML.

Verifique se está funcionando

Depois de aplicar essas soluções, acesse seu site novamente:

https://seuusuario.github.io

Se ainda não funcionar, tente limpar o cache do navegador (Ctrl+F5).

Configuração Avançada (se necessário)

Criar arquivo de configuração

Às vezes, é necessário adicionar um arquivo de configuração para garantir que o GitHub Pages reconheça seu site:

Crie um arquivo chamado .nojekyll (sem extensão) na raiz do seu repositório.

Este arquivo diz ao GitHub para não usar o processador Jekyll, que às vezes causa problemas com sites HTML simples.

Verificar URL correta

Certifique-se de que está acessando a URL correta:

https://seunomedeusuario.github.io (não é https://github.com/seunomedeusuario/seunomedeusuario.github.io)

Muitas pessoas confundem a URL do repositório com a URL do site publicado.

Último recurso

Se nada funcionar, tente estas alternativas:

  1. Exclua e recrie o repositório
  2. Use o Netlify Drop (arraste e solte o arquivo HTML)
  3. Use o Vercel para implantação
